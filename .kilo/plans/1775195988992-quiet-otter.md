## 1. Downloader Race Condition (downloader/app.py:98-168)

**Problem**: sha256 check and INSERT are not atomic - concurrent runs could corrupt data.

**Fix**: Wrap the check + insert in a single transaction with a single cursor.

**Approach**:
```python
# Replace lines 95-169 with:
h = sha256(path)
cur = db.cursor()
try:
    cur.execute("SELECT file_path FROM media WHERE sha256=%s", (h,))
    existing = cur.fetchone()
    
    if existing:
        os.remove(path)
        path = existing[0]
    else:
        thumb = make_thumb(path)
        cur.execute("""
           INSERT INTO media(post_id,url,file_path,thumb_path,sha256,downloaded_at,status)
           VALUES(%s,%s,%s,%s,%s,%s,'done')
           ON CONFLICT DO NOTHING
        """, (post_id, url, path, thumb, h, datetime.utcnow()))
    db.commit()
finally:
    cur.close()
```

## 5. Comments tsvector Trigger Missing (db/init.sql)

**Problem**: `comments.tsv` column exists but never populated.

**Fix**: Add after line 36:
```sql
CREATE FUNCTION comments_tsv_trigger() RETURNS trigger AS $$
BEGIN
  NEW.tsv := to_tsvector('english', coalesce(NEW.body,''));
  RETURN NEW;
END $$ LANGUAGE plpgsql;

CREATE TRIGGER comments_tsv_update
BEFORE INSERT OR UPDATE ON comments
FOR EACH ROW EXECUTE FUNCTION comments_tsv_trigger();
```
